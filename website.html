Vibey

Build with words, not code.

For those who don't code but want to use AI to:

Create workflows to automate repeated tasks.
Perform research in the background.
Explore product ideas and iterate quickly.
Create small games from a simple prompt.
Create teaching materials tailored to your needs.
Run ad-hoc analysis and create visualizations.

How is vibey different?

Your description of what you're building. The dialogs with AI while building it. How you orchestrate your agents. Documents are the source of truth for everything. There is no database.

Your documents are not only text: use images, audio, and embed small apps in your documents. No terminal or dedicated native app required.

The agents don't ask for permission, they just run the commands that they need for the task you give them, so they work at full speed. But each project is fully isolated in its own container and volume. A rogue agent's blast radius is limited to its own project — it cannot touch other projects, vibey, or your computer.

Bring your own OpenAI/Anthropic credentials, whether personal or API keys.

Both the local and cloud versions are open source.

Choose your own adventure

After reading the five things above, choose how you want to run vibey.

Run vibey locally

The default path: everything runs on your machine. You bring your own AI credentials, keep full control, and get started with Docker only.

1. Install
curl -fsSL https://raw.githubusercontent.com/altocodenl/vibey/main/install.sh | sh
2. Start
cd vibey docker compose up --build
3. Open
http://localhost:5353
Stop vibey
docker compose down
Reset all data
docker compose down -v

Use vibey on the cloud

Cloud mode is for when you want your agents to keep working while you are away, open your projects from any device, and share them with others.

  • Always running — your agents can work while your computer is closed.
  • Available from any device — all you need is a browser.
  • Shareable — publish and share projects with others.
  • Zero lock-in — the whole thing is open source, so you can always run it yourself.
